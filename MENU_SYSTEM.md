# 动态菜单说明

## 概述

系统已实现根据菜单 `path` 字段自动区分管理端和用户端菜单的功能,不再依赖硬编码的菜单ID。

## 菜单分类规则

### 管理端菜单 (`/admin/Layout.vue`)
- **筛选条件**:
  1. 排除 `path` 以 `/user/` 开头的菜单
  2. `menuType !== 2` (排除按钮权限)

- **实现位置**: `src/admin/Layout.vue` - `loadMenus()` 函数

### 用户端菜单 (`/user/UserLayout.vue`)
- **筛选条件**:
  1. `path` 必须以 `/user/` 开头
  2. `path` 不为空
  3. `menuType !== 2` (排除按钮权限)

- **实现位置**: `src/user/UserLayout.vue` - `loadMenus()` 函数

## menuType 字段说明

- **0**: 目录
- **1**: 菜单
- **2**: 按钮(权限标识,不参与菜单渲染)

## 数据流转

1. 从后端获取所有菜单数据 (`getAllMenus()`)
2. 根据 `path` 前缀筛选对应端口的菜单
3. 过滤掉按钮类型 (`menuType=2`) 的菜单
4. 过滤掉没有 `path` 的空菜单(仅用户端)
5. 将平铺的菜单数据转换为树形结构 (`buildMenuTree()`)
6. **自动过滤空目录**: 如果目录(menuType=0)没有子菜单,自动从菜单树中移除
7. 通过 `DynamicMenu` 组件递归渲染

## DynamicMenu 组件

- **位置**: `src/components/DynamicMenu.vue`
- **功能**: 递归渲染菜单树,支持子菜单和单菜单项
- **支持的图标**: 已映射 Element Plus 所有图标组件

## 空目录过滤机制

系统会在构建菜单树时自动过滤掉没有子菜单的空目录：

1. **目录类型(menuType=0)**:
   - 如果有子菜单,递归过滤子菜单
   - 如果过滤后子菜单为空,则移除该目录
2. **菜单类型(menuType=1)**: 无论是否有子菜单,都会保留
3. **递归过滤**: 从顶级目录开始,递归检查每一层,确保显示的目录都包含实际的菜单项

## 注意事项

1. 新增菜单时,确保 `path` 字段正确设置:
   - 管理端菜单: 任何不以 `/user/` 开头的有效路径(如 `/admin/模块/功能` 或 `/模块/功能`)
   - 用户端菜单: 必须以 `/user/` 开头(如 `/user/模块/功能`)

2. 按钮权限菜单的 `menuType` 必须设置为 `2`,并且设置 `perms` 权限标识

3. 系统会自动缓存菜单数据到 `userStore.menus`,避免重复请求

4. 用户端菜单必须有 `path` 字段,没有 `path` 的菜单会被过滤掉

5. 后端返回的菜单只要不是 `/user/` 开头都会被归类为管理端菜单
